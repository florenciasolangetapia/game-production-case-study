# Production Roadmap

**Project:** La Última Tanda  
**Owner:** Florencia Solange Tapia  
**Target:** Playable vertical slice  
**Timeline:** 16 weeks  
**Status:** Planning baseline

## Planning Assumptions

- Six-person cross-functional team
- Two-week production cycles
- Remote-first collaboration
- PC as the initial target platform
- No dedicated QA role during early production
- Final four weeks reserved for testing, fixes, and polish

## Milestone Plan

| Milestone | Timeline | Primary Outcome | Exit Criteria |
|---|---:|---|---|
| **M0 — Pre-production Alignment** | Weeks 1–2 | Scope and production approach agreed | Brief approved, backlog structured, major risks identified, and responsibilities assigned |
| **M1 — Core Loop Prototype** | Weeks 3–5 | Basic order-to-delivery loop is playable | Player can receive, prepare, and deliver one order using placeholder assets |
| **M2 — First Playable** | Weeks 6–9 | Complete shift playable from start to finish | One full shift, three customers, basic dialogue, time management, and end-of-shift result |
| **M3 — Content Complete Alpha** | Weeks 10–12 | Vertical-slice content integrated | Four recipes, final dialogue content, interface pass, audio hooks, and tracked known defects |
| **M4 — Playtest Build** | Weeks 13–14 | Stable external testing candidate | No blocker defects, test instructions prepared, feedback method agreed, and build distributed |
| **M5 — Release Candidate** | Weeks 15–16 | Portfolio-ready vertical slice | Acceptance criteria met, known issues documented, final build reviewed, and retrospective completed |

## Production Cadence

- **Weekly planning:** priorities, ownership, and dependencies
- **Short daily check-in:** progress, next action, and blockers
- **Weekly build review:** playable progress and quality assessment
- **Risk review:** updated at least once per week
- **Milestone review:** formal approval against exit criteria
- **Retrospective:** completed after each major milestone

## Scope Protection

To protect the 16-week timeline:

- The vertical-slice scope is locked after M2.
- New ideas are added to a future-opportunities list instead of entering active production.
- Must-have work is prioritized before polish and optional content.
- The final four weeks are not used to introduce new features.
- Scope changes require a documented impact assessment.

## Key Dependencies

| Dependency | Impact | Production Response |
|---|---|---|
| Core loop before final art | Art may be created for unvalidated interactions | Validate gameplay using placeholder assets |
| Recipe list before UI implementation | Interface may require repeated rework | Approve recipe requirements during pre-production |
| Dialogue structure before audio planning | Voice and sound requirements remain unclear | Lock the vertical-slice dialogue format by M2 |
| Stable build before playtesting | Test results may be affected by technical failures | Complete blocker triage before external testing |
| Acceptance criteria before QA | Quality decisions become subjective | Define completion criteria at backlog-item level |

## Roadmap Review Rule

A milestone is only considered complete when its exit criteria are met. Calendar dates alone do not determine completion.
